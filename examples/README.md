# examples/

Worked, runnable illustrations of the system for humans: sample inputs, sample outputs,
demo JSON payloads, and step-by-step walkthroughs.

Status: intentionally empty at foundation stage - examples are added only for interfaces
that exist and are validated, so examples never teach unverified behaviour.

Rules:
- Every example states the engine/contract version it was produced with.
- Example outputs are regenerated (never hand-edited) whenever the producing version changes.
- Examples are illustrative, not normative: tests and specifications govern behaviour.

Planned contents (placeholders, per audit fix 2): sample charts, example inputs, example
outputs, demo JSON, walkthrough examples.
