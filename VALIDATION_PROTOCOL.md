# Validation Protocol

A tier can be LOCKED only when all applicable conditions pass:

1. Formal specification/profile is frozen.
2. Executable source exists.
3. Dependencies and versions are pinned.
4. Automated tests exist.
5. Calibration/debug fixtures are separated from untouched holdouts.
6. Anti-fitting scan passes.
7. Machine-readable evidence is generated by the live run.
8. Human-readable certification report is generated from the same evidence.
9. Fresh extraction reproduces the result with the documented command.
10. The repository lock manifest is updated only after evidence passes.

A failed tier remains NOT LOCKED. The exact failing layer and residual/mismatch must be recorded.
