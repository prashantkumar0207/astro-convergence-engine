# docs/ - Governing Documents

Normative layer of the repository. If code and a ratified document disagree, the document
governs until a decision log entry says otherwise.

| Document | Role |
|---|---|
| `MASTER_ARCHITECTURE.md` | System shape, boundaries, dependency rules |
| `PROJECT_ROADMAP.md` | Phased plan, milestones, exit criteria |
| `DECISION_LOG.md` | Append-only record of binding decisions (ADR style) |
| `HLKG_SPEC.md` | Specification: HLKG (expansion pending - Q4) |
| `AKG_SPEC.md` | Specification: AKG (expansion pending - Q4) |
| `VALIDATION_STANDARD.md` | What counts as proof in this project |
| `DOCUMENTATION_STANDARD.md` | How documents are written, versioned, reviewed |
| `OPEN_QUESTIONS.md` | Register of ambiguities awaiting owner decisions |

Editing rules: see `DOCUMENTATION_STANDARD.md`. Every document carries a status header.
