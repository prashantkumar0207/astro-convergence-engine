# Astro Convergence Engine
# Engineering Constitution

Version: 1.0.0

Status: ACTIVE

---

# Purpose

This constitution defines the engineering principles that govern the Astro Convergence Engine repository.

Every contributor—human or AI—must follow this document.

This constitution has higher priority than implementation prompts.

---

# Principle 1 — Determinism

Every calculation must be reproducible.

Given identical inputs, the engine must always produce identical outputs.

Random behaviour is prohibited unless explicitly documented.

---

# Principle 2 — Explainability

Every prediction must be explainable.

The engine must be able to answer:

Why?

Using which rules?

Using which evidence?

Using which astrology systems?

---

# Principle 3 — Knowledge First

Knowledge belongs in structured knowledge files.

Business logic belongs in engine modules.

Knowledge must never be hardcoded into algorithms.

---

# Principle 4 — Separation of Concerns

Every module has one responsibility.

No module may perform responsibilities owned by another module.

---

# Principle 5 — Immutable Knowledge

Knowledge becomes read-only after loading.

Runtime code must never modify canonical knowledge.

---

# Principle 6 — Validation Before Execution

Nothing enters the engine without validation.

---

# Principle 7 — Stable Public APIs

Public APIs should evolve carefully.

Breaking changes require explicit versioning.

---

# Principle 8 — Documentation

Every public module requires documentation.

Every public function requires documentation.

---

# Principle 9 — Testing

Every new feature requires deterministic tests.

No feature is considered complete without tests.

---

# Principle 10 — Git

Every commit must describe one logical change.

Large unrelated commits are prohibited.

---

# Principle 11 — Reviews

Every implementation generated by AI must be reviewed before entering Git.

---

# Principle 12 — Long-Term Maintainability

Readability is preferred over cleverness.

Maintainability is preferred over micro-optimisation.

The repository is expected to live for many years.

---

# End of Constitution
